📋 课程总览

课程一：悟空·硅基生产力引擎（2天版）

本课程面向企业技术负责人与IT骨干，以“谋定而后动、简约之为、精准施策、目标牵引”为核心理念。通过两天密集训练，帮助学员理解悟空的四层架构（安全层、模型层、编排层、生态层），掌握六大扩展手段（API/MCP/RPA/Skills/Plugin/Tools）的选型原则，并现场产出一套可落地的企业级应用方案。

  • 核心产出：一套可执行的企业级悟空落地方案 + 若干可直接部署的 Skill 资产。
  • 适合人群：具备基本 API 概念，了解企业数字化现状的技术决策者。

课程二：技术架构师的全链路 Agent 系统课（3天版）

这是专为技术架构师设计的深度技术版课程。在两天版基础上增加“魔改架构、多Agent集群、自主能力演进”等高阶内容。课程不仅讲“悟空能做什么”，更深入讲“悟空为什么这样做”以及“如何让它为你做”。

  • 核心产出：一套可落地的企业级悟空定制化方案 + 多Agent协同架构设计。
  • 适合人群：具备系统设计和 API 开发扎实基础，理解企业级 AI 落地完整技术栈的架构师。

📖 详细课程大纲

课程一：悟空·硅基生产力引擎（2天）

DAY1：架构解密与能力扩展

模块一：技术视角下的悟空：开场+架构与边界

  • 痛点共鸣：从“业务痛”到“技术方案”的思维切换
  • 悟空整体架构解密：四层模型（安全/模型/编排/生态）
  • 安全沙箱与数据隔离机制：身份/数据/操作三层沙箱
  • 模型接入与能力边界：Model Gateway 与智能路由策略

模块二：悟空六大扩展手段：原理与选型

  • 五种扩展手段全景对比：API/MCP/RPA/Skills/Plugin/Tools
  • Skills：最小可用封装单元（触发词+业务规则+验收标准）
  • MCP：跨系统数据管道（Resources/Tools/Prompts）
  • 三阶选型决策树：根据业务需求选择最优扩展路径

DAY2：技能封装与场景共创

模块三：系统打通专题：RPA/MCP/API 三路并进

  • 数据孤岛诊断：接口孤岛/协议孤岛/界面孤岛
  • RPA：无接口系统的兜底打通方案
  • MCP vs API：何时选谁，何时组合

模块四：技能封装实操：从概念到可运行

  • 降级 Skill 封装实操：AI不可用时的兜底方案设计
  • 场景共创与技术可行性评估
  • 方案路演与架构级点评

课程二：技术架构师的全链路 Agent 系统课（3天）

DAY1：架构解密与六维扩展

模块一：技术人专属痛点共鸣与ROI速算

  • 从“业务用户思维”切换到“系统集成商思维”
  • 技术就绪度自测：API理解/MCP概念/安全认知

模块二：悟空架构深度解密

  • 安全沙箱三层架构：身份/数据/操作沙箱的底层逻辑
  • 模型网关与路由策略：多模型热切换与降级链设计

模块三：六维扩展手段深度解析

  • Skills：业务能力封装的标准化模板
  • MCP：跨系统通信协议的源码级理解
  • API：系统级深度集成的避坑指南

DAY2：系统打通与技能封装

模块四：数据孤岛三路打通方案

  • RPA：界面级流程自动化的配置与容错
  • 多模态数据源组合策略：API+RPA+MCP 联合查询

模块五：生产级 Skill 封装实战

  • 从概念到可运行：需求冻结→触发词设计→业务规则编写
  • 降级机制设计：AI不可用时的 SOP 自动输出

DAY3：高阶架构与自主进化

模块六：悟空架构魔改与多Agent集群

  • 魔改架构：如何根据企业需求定制悟空底层逻辑
  • 多Agent协同：Planning + Memory + Tools 的深度编排

模块七：自主能力演进与方案发布

  • 自主进化：如何让 Agent 从数据反馈中持续迭代
  • 行动计划发布：90天技术追踪与资源包交付

🎯 Demo场景简介

本课程包含多个高价值实战场景，覆盖从业务封装到系统打通的核心环节：

✅ 客服工单智能分拣与降级 Skill 封装

展示如何用Skill而非代码解决高频业务问题，实现AI不可用时自动输出分拣SOP，保证业务不中断。

演示如何通过MCP协议从第三方ERP拉取实时订单数据，并在悟空中完成复杂条件筛选与分析。

针对无API的老旧餐饮POS系统，通过RPA模拟人工点击完成日结数据导出，解决界面孤岛难题。

✅ 跨系统客户画像一键生成

融合POS、会员、外卖、ERP四个异构数据源，利用API+RPA组合方案替代人工跨系统导数。

✅ 多Agent协同：销售助手集群

展示Planning Agent如何拆解任务，调度Research Agent和Drafting Agent协同完成竞品分析报告。
