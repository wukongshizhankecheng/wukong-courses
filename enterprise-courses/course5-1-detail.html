📋 课程总览

课程定位

《悟空赋能银行业务增长》是一门专为银行等金融机构设计的 AI 落地实操课程。课程从"大语言模型是什么、智能体是什么"出发，帮助学员建立清晰的 AI 认知底座，随后聚焦一线人员每天真实面对的高频工作任务，通过讲师演示和分组实战，带领学员深度体验客户画像、理财产品解读、合规话术生成、投研晨会材料、客户投诉处理五类核心工作流。

课程目标
  • 清晰区分大语言模型（LLM）与 AI 智能体（Agent）的概念与边界。
  • 掌握将业务任务转化为悟空指令的基本方法（定角色、给资料、说目标、定格式、控边界、验结果）。
  • 深度体验客户经营、合规话术、投研摘要、客户服务等典型任务，并输出结构化成果文档。
  • 识别本岗位真实工作中的候选 AI 应用场景，形成可落地的行动计划。

💡 本课程强调"认知先行、合规优先、成果导向"，帮助一线人员从"凭经验服务客户"转向"基于结构化资料经营客户"，在合规前提下实现工作效率的显著提升。

📖 详细课程大纲

模块一：从大语言模型到智能体（09:00—10:30）

第一节：大语言模型（LLM）是什么

  • LLM 的本质：基于海量文本训练的概率预测系统
  • LLM 能做什么：文本摘要、文章生成、语言改写、问题回答
  • LLM 不能做什么：主动获取实时信息、调用外部系统、保证事实绝对准确
  • 银行场景的直接启示：LLM 可以帮你写初稿、整理资料，但不能替你做授信判断

第二节：AI 智能体（Agent）是什么

  • 从 LLM 到 Agent 的跨越：给大脑加上"眼睛、手脚和记忆"
  • Agent 的四个核心要素：感知 → 规划 → 行动 → 反馈
  • Agent 的关键能力：工具调用、记忆管理、多步骤推理、自主纠错
  • 互动讨论：Agent 与 LLM 在银行场景中的本质区别

第三节：智能体在各个行业的真实应用

  • 智能体应用全览：教育、电商、生产制造、财税法务等行业场景
  • 银行业已有的 Agent 应用类型：智能客服、合规审核、投研助手、贷前调查
  • 悟空在银行业务中的定位：企业级 AI 智能体，与钉钉工作场景深度集成

模块二：悟空产品介绍与 Skill 入门（10:30—11:15）

悟空基础使用

  • 悟空的访问方式：钉钉 PC 端、移动端的入口与界面介绍
  • 对话模式与文档模式的区别与适用场景
  • 上传文件、引用资料、多轮对话的基本操作

任务指令设计六步法

  • ① 定角色：告诉悟空它是谁（例："你是银行合规销售话术教练"）
  • ② 给资料：上传或粘贴输入材料
  • ③ 说目标：明确要完成什么任务
  • ④ 定格式：要求输出格式（例："以表格形式，每行一个场景"）
  • ⑤ 控边界：明确不能做什么（例："不得承诺收益，不得弱化风险"）
  • ⑥ 验结果：检查输出是否合规可用

Skill 定制逻辑入门

  • Prompt 与 Skill 的区别：一次性指令 vs 可复用的工作流
  • 哪类银行任务适合做成 Skill：客户画像生成、产品说明摘要、合规话术检查等
  • Skill 的训练流程简介：确定场景 → 准备数据 → 多轮调试 → 人工验收 → 封装上线

模块三：客户经营案例深度演示（11:15—12:00）

演示场景背景

  • 某支行零售客户经理小王负责服务一位 52 岁的私营业主客户，需求不明确。
  • 小王手头有客户的基础资料、近三次沟通纪要和当前产品持有记录。

演示内容与流程

  • Step 1 — 准备输入资料：展示客户基础资料、沟通纪要和产品持有记录
  • Step 2 — 构建任务指令：现场使用任务指令六步法构建完整指令
  • Step 3 — 执行与观察：实时展示悟空的执行过程，解释如何理解输入、提取信息
  • Step 4 — 审核与优化：展示生成结果，逐项进行人工审核，演示如何通过追问指令优化输出
  • Step 5 — 成果展示：展示最终生成的《客户画像与需求分析表》《客户跟进计划》《客户沟通建议话术》

模块四：场景实战工作坊——四大案例实战（13:30—16:00）

案例 A：理财 / 贷款产品说明摘要与 FAQ 生成

  • 场景背景：新上线一款 180 天期限的净值型理财产品，说明书 28 页，需在两天内掌握要点。
  • 实操任务：生成《产品说明卡》《产品卖点与风险点对照表》《客户 FAQ》《客户沟通注意事项》。
  • 核心指令要点：不夸大产品收益或保障范围；不弱化风险提示；不替代正式产品说明书。

案例 B：合规销售话术与风险提示语生成

  • 场景背景：支行因客户投诉"理财经理承诺了保本"被合规部门约谈，需统一合规话术。
  • 实操任务：生成《合规销售话术》《风险提示语》《客户异议回应模板》《高风险表达替代表》。
  • 核心指令要点：不承诺收益；不保证本金安全；不弱化风险提示；不生成诱导性销售表达。

案例 C：投研摘要与晨会材料生成

  • 场景背景：财富管理团队每天早上需要准备晨会材料，传统方式需要 40—60 分钟准备。
  • 实操任务：生成《投研摘要》《晨会分享材料》《市场周报初稿》《投资者教育提示语》。
  • 核心指令要点：只做资料整理和内部学习辅助；不生成买入、卖出、持有等投资建议。

案例 D：客户投诉分类与回复建议生成

  • 场景背景：客服中心收到一封客户投诉邮件，需在 2 小时内完成投诉分类、诉求归纳和回复建议初稿。
  • 实操任务：生成《投诉分类表》《客户诉求归纳表》《回复建议初稿》《投诉复盘要点》。
  • 核心指令要点：回复建议要有同理心、清晰、稳妥；不自动认定机构责任；不自动承诺赔付。

模块五：场景共创与优先级排序（16:15—17:15）

第一步：候选场景提报

  • 各小组基于上午的认知建立和下午的实战体验，思考并回答以下问题：
  • 我们岗位上哪些工作是高频、重复、重文档的？
  • 哪些工作的输入资料是可以整理和上传的？
  • 哪些工作的输出结果有相对固定的格式？
  • 哪些工作的风险边界是清晰的？

第二步：MoSCoW 优先级排序

  • Must Have（必须做）：高频 + 高价值 + 低风险 + 资料易获取
  • Should Have（应该做）：高价值但实施难度中等，或资料需要整理
  • Could Have（可以做）：有价值但优先级较低，或风险边界需进一步明确
  • Won't Have（暂不做）：风险边界不清晰、资料无法获取、或涉及核心判断不宜 AI 介入

第三步：机构候选清单分析

  • 讲师带领全班汇总排序结果，形成《机构候选 Skill 场景优先级清单》。
  • Must Have 类场景是后续 Skill 训练营的优先选题。

模块六：课程小结与行动计划（17:15—17:30）

讲师 5 分钟总结

  • 认知层：LLM 是生成能力，Agent 是执行能力，悟空是工作能力。
  • 操作层：任务指令六步法是今天最重要的工具。
  • 行动层：今天识别出来的 Must Have 场景，就是下一步最值得推进的方向。

🎯 Demo场景简介

本课程包含5个实战场景Demo，覆盖银行业务核心场景：

✅ 客户画像与需求分析生成

将散落的沟通纪要、产品持有记录整合为结构化画像，10分钟内完成原本需30-60分钟的工作。

✅ 理财产品说明摘要与FAQ生成

基于28页产品说明书，20分钟生成产品说明卡、卖点风险对照表及客户FAQ，统一沟通口径。

✅ 合规销售话术与风险提示语生成

针对六类敏感场景生成合规话术初稿，将"从零编写"变为"从初稿优化"，大幅缩短标准化周期。

✅ 投研摘要与晨会材料生成

自动完成投研报告和市场信息的摘要整理，10-15分钟生成晨会材料，节省主管40-60分钟准备时间。

✅ 客户投诉分类与回复建议生成

快速完成投诉信息的结构化整理，10-15分钟生成回复建议初稿，让主管精力集中在责任判断上。
